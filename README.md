# FinEdge Backend

A personal finance management API built with Node.js and Express. FinEdge helps users track income and expenses, view spending trends, and get AI-powered financial advice.

## Getting Started

### Prerequisites

- Node.js (v18 or higher)
- MongoDB
- npm or yarn

### Installation

```bash
# Install dependencies
npm install

# Create a .env file (see Environment Variables section)
cp .env.example .env

# Start the server
npm run dev    # Development (with hot reload)
npm start      # Production
```

### Health Check

```bash
GET http://localhost:3000/health
```

Response: `{ "status": "ok" }`

## API Reference

**Base URL:** `http://localhost:3000/api`

All routes under `/api/v1/*` (except signup and login) require authentication. Include the JWT token in the `Authorization` header:

```
Authorization: Bearer <your_access_token>
```

---

### 1. Users Module (`/api/v1/users`)

#### Sign Up (Create Account)

Create a new user account.

| Method | Endpoint | Auth Required |
|--------|----------|---------------|
| POST | `/api/v1/users/signup` | No |

**Request Body:**

```json
{
  "name": "John Doe",
  "email": "john@example.com",
  "password": "secret123"
}
```

| Field | Type | Required | Rules |
|-------|------|----------|-------|
| name | string | Yes | Min 2 characters |
| email | string | Yes | Valid email format |
| password | string | Yes | Min 6 characters |

**Response:** User object (id, name, email â€” password excluded)

---

#### Login

Log in and receive JWT tokens.

| Method | Endpoint | Auth Required |
|--------|----------|---------------|
| POST | `/api/v1/users/login` | No |

**Request Body:**

```json
{
  "email": "john@example.com",
  "password": "secret123"
}
```

**Response:**

```json
{
  "user": { "id": "...", "name": "...", "email": "..." },
  "accessToken": "eyJhbGc...",
  "refreshToken": "eyJhbGc..."
}
```

Use `accessToken` in the `Authorization` header for protected routes.

---

#### Get Current User

Get the logged-in user's profile.

| Method | Endpoint | Auth Required |
|--------|----------|---------------|
| GET | `/api/v1/users/` | Yes |

**Response:** User object (id, name, email, preferences)

---

#### Update Preferences

Update user preferences (e.g., categories of interest).

| Method | Endpoint | Auth Required |
|--------|----------|---------------|

|-------|------|----------|

### 2. Transactions Module (`/api/v1/transactions`)

All transaction endpoints require authentication.

#### Create Transaction

Add a new income or expense. The **category** is auto-generated by AI from the title.

| Method | Endpoint | Auth Required |
|--------|----------|---------------|
| POST | `/api/v1/transactions` | Yes |

**Request Body:**

```json
{
  "type": "expense",
  "amount": 50.00,
  "date": "2025-03-09T10:00:00.000Z",
  "title": "Lunch at cafe",
  "notes": "Optional notes"
}
```

| Field | Type | Required | Rules |
|-------|------|----------|-------|
| type | string | Yes | `"income"` or `"expense"` |
| amount | number | Yes | Positive number |
| date | string | Yes | ISO 8601 datetime |
| title | string | Yes | Transaction description |
| notes | string | No | Max 100 chars |

**Response:** Created transaction (includes auto-generated `category`)

---

#### Get All Transactions

List transactions with optional filters and pagination.

| Method | Endpoint | Auth Required |
|--------|----------|---------------|
| GET | `/api/v1/transactions` | Yes |

**Query Parameters:**

| Param | Type | Required | Description |
|-------|------|----------|-------------|
| page | number | No | Page number (pagination) |
| limit | number | No | Items per page (max 10) |
| type | string | No | Filter by `income` or `expense` |
| category | string | No | Filter by category |
| startDate | string | No | ISO datetime â€” filter from date |
| endDate | string | No | ISO datetime â€” filter to date |

**Example:**

```
GET /api/v1/transactions?page=1&limit=5&type=expense
```

**Response:**

```json
{
  "transactions": [...],
  "pagination": {
    "total": 25,
    "totalPages": 5,
    "page": 1,
    "limit": 5,
    "hasNext": true,
    "hasPrev": false
  }
}
```

---

|--------|----------|---------------|

#### Get Transaction Summary

Get aggregated income, expense, balance, and transaction count for a date range.

| Method | Endpoint | Auth Required |
|--------|----------|---------------|
| GET | `/api/v1/transactions/summary` | Yes |

**Query Parameters:**

| Param | Type | Required | Description |
|-------|------|----------|-------------|
| startDate | string | No | ISO datetime (default: 30 days ago) |
| endDate | string | No | ISO datetime (default: today) |

**Example:**

```
GET /api/v1/transactions/summary?startDate=2025-02-01T00:00:00.000Z&endDate=2025-03-09T23:59:59.000Z
```

**Response:**

```json
[
  {
    "income": 5000,
    "expense": 3200,
    "balance": 1800,
    "transactions": 45
  }
]
```

---

#### Get Transaction Trend

Get daily income, expense, and balance over time (for charts).

| Method | Endpoint | Auth Required |
|--------|----------|---------------|
| GET | `/api/v1/transactions/trend` | Yes |

**Query Parameters:** Same as summary (`startDate`, `endDate`)

**Response:**

```json

  { "date": "2025-03-02", "income": 0, "expense": 25, "balance": -25, "transactions": 1 }

```

### 3. Budget Module (`/api/v1/budget`)

Set and fetch monthly budget goals and savings targets. One budget per user per month.

All budget endpoints require authentication.

#### Get Budget

Fetch budget for the current month or a specific month.

| Method | Endpoint | Auth Required |
|--------|----------|---------------|
| GET | `/api/v1/budget` | Yes |

**Query Parameters:**

| Param | Type | Required | Description |
|-------|------|----------|-------------|
| month | string | No | Month in `YYYY-MM` format (default: current month) |

**Example:**

```
GET /api/v1/budget

```

**Response (when budget exists):**

```json
{
  "id": "...",
  "user": "...",
  "month": "2025-03",
  "monthlyGoal": 5000,
  "savingsTarget": 1500
}
```

**Response (when no budget set):**

```json
{
  "monthlyGoal": 0,
  "savingsTarget": 0,
  "month": "2025-03"
}
```

## Authentication

1. **Sign up** or **login** to get `accessToken` and `refreshToken`.
2. For protected routes, add the header:
   ```
   Authorization: Bearer <accessToken>
   ```
3. If the token is missing or invalid, you'll receive `401 Unauthorized`.

## Environment Variables

Create a `.env` file in the project root:

| Variable | Description | Example |
|----------|-------------|---------|
| `PORT` | Server port | `3000` |
| `MONGODB_URI` | MongoDB connection string | `mongodb://localhost:27017/finedge` |
| `JWT_SECRET` | Secret for signing JWT tokens | `your-secret-key` |
| `JWT_EXPIRES_IN` | Access token expiry | `1h` |
| `JWT_REFRESH_EXPIRES_IN` | Refresh token expiry | `7d` |
| `CORS_ORIGIN` | Allowed origins (comma-separated) | `http://localhost:5173` |
| `GITHUB_TOKEN` | API key for AI service (GitHub inference) | Required for Suggest and auto-categorization |

## Quick Start Example

```bash
# 1. Sign up
curl -X POST http://localhost:3000/api/v1/users/signup \
  -H "Content-Type: application/json" \
  -d '{"name":"Jane","email":"jane@example.com","password":"pass1234"}'

# 2. Login
curl -X POST http://localhost:3000/api/v1/users/login \
  -H "Content-Type: application/json" \
  -d '{"email":"jane@example.com","password":"pass1234"}'

# 3. Create a transaction (use the accessToken from login)
curl -X POST http://localhost:3000/api/v1/transactions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN" \
  -d '{"type":"expense","amount":25,"date":"2025-03-09T12:00:00.000Z","title":"Coffee"}'

# 4. Get summary
curl "http://localhost:3000/api/v1/transactions/summary" \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN"

# 5. Set budget (monthly goal & savings target)
curl -X PUT http://localhost:3000/api/v1/budget \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN" \
  -d '{"monthlyGoal":5000,"savingsTarget":1500}'

# 6. Get budget
curl "http://localhost:3000/api/v1/budget" \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN"
```

## Swagger/OpenAPI Docs

- Local API docs UI: `http://localhost:3000/docs`
- OpenAPI JSON endpoint: `http://localhost:3000/docs/openapi.json`
- Static docs files for GitHub Pages are generated into `/docs`.

Generate the static OpenAPI spec:

```bash
npm run swagger:generate
```

### GitHub Pages deployment

1. Commit the `/docs` folder (`index.html` + `openapi.json`).
2. In GitHub repository settings, enable **Pages**.
3. Set source to:
   - Branch: `main`
   - Folder: `/docs`

### Automatic deployment on merge to `main`

This repository includes a GitHub Actions workflow at `.github/workflows/swagger-pages.yml` that runs on every push/merge to `main`:

1. Regenerates `docs/openapi.json` using `node generateSwagger.js`
2. Uploads the `/docs` directory as a Pages artifact
3. Deploys it to GitHub Pages
